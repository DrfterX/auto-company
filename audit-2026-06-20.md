## 3. 跨项目分析

### 3.1 重复代码模式

7 个子项目中 monito 和 statushub 都是 **Cloudflare Workers + TypeScript** 技术栈，结构类似（wrangler.toml + src/ + migrations/）。这本身合理（同类产品用同栈），但**没有看到共享的 wrangler/TS 模板**——意味着每个项目各自维护 boilerplate。

### 3.2 命名/风格不一致

- `options_arbitrage_system` 是 Python 主导（核心交易系统）
- `monito / statushub / critiq / snapog` 是 TypeScript/Cloudflare Workers
- `bot` 是 Python（与 options_arbitrage_system 配套）
- `monito-checker` 是 TS

这种 "Python 后端 + TS 前端/Workers" 分工合理。

### 3.3 共享依赖情况

未发现明显的 `shared/` 或 `common/` 跨项目目录。Python 项目都在各自 .venv 里管依赖（`uv.lock` 300KB 在 options_arbitrage_system）。

### 3.4 配置漂移

未深入比较配置文件。但 `.env.template` 在 options_arbitrage_system 里有（476 字节），monito 有 `.env`（437 字节），monito-checker 等没有明确 env 配置——**可能存在配置项定义不一致**。
