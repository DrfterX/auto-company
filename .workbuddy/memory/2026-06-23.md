## 修复 sensenova-proxy Key 管理脱节（09:46）

### 根因
proxy 的 `next_key()` 使用全局 `_key_index` 计数器，完全忽略 auto-loop.sh 通过 `ANTHROPIC_AUTH_TOKEN` 指定的 key。导致两边各自管理 key 轮换，互不知晓对方状态。

### 修改
**`sensenova-proxy.py`:**
1. `next_key(preferred=None)` — 新增 preferred 参数，设置时重置 _key_index 到对应位置
2. 请求处理中使用 `ANTHROPIC_AUTH_TOKEN` 作为 preferred key
3. all-exhausted 响应增加 `all_exhausted` flag

**`auto-loop.sh`:**
4. 429 处理增加 proxy all-exhausted 检测（grep RESULT_MESSAGE），避免 transient guard 误吞
5. 更新 transient guard 条件：仅 cost=0 且非 all-exhausted 时触发
